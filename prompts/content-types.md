# 小红书图文内容类型 · 布局模板

> 本文件定义不同类型小红书图文的**内容结构**。
> 视觉风格见 `style-templates.md`，两者组合使用：
> **内容类型（本文件）** × **视觉风格（style-templates.md）** = 最终页面

---

## 内容类型一览

| 类型 | 典型标题格式 | 页面结构 | 页数 |
|------|------------|---------|------|
| 教程步骤型 | "3步搞定XX" / "手把手教你XX" | 封面+步骤页×N+末页 | 5-8 |
| 合集推荐型 | "5个XX推荐" / "N个必备XX" | 封面+单品页×N+末页 | 5-10 |
| 对比型 | "XX vs XX" / "用了XX之后" | 封面+对比页×N+结论页 | 4-6 |
| 清单型 | "XX的10个注意事项" | 封面+清单页(每页3-5条)+末页 | 4-6 |
| 金句/语录型 | "一句话XX" / 纯文字力量 | 封面+金句页×N+末页 | 3-5 |
| 日报资讯型 | "今日N条XX" / "本周XX盘点" | 封面+资讯页×N+末页 | 4-7 |
| 数据报告型 | "XX数据告诉你" / "N%的人不知道" | 封面+数据页×N+结论页 | 4-6 |
| 故事/经验型 | "我如何XX" / "踩坑N年后" | 封面+叙述页×N+总结页 | 5-8 |
| 工具推荐型 | "XX神器推荐" / "效率提升N倍" | 封面+工具页×N+末页 | 5-8 |
| 测评/种草型 | "XX真实测评" / "用了N天后" | 封面+维度评分页+末页 | 5-7 |

## 教程步骤型

### 适用场景
编程教程、操作指南、DIY教程、烹饪步骤、化妆教程

### 页面结构
```
Page 1 — 封面
├── 大标题（痛点/好处）
├── 副标题（"只需N步"）
└── 步骤数量预告标签

Page 2~N-1 — 步骤页（每页一步）
├── 步骤编号（超大装饰数字）
├── 步骤标题（一句话动作）
├── 步骤说明（≤2行）
└── 代码/截图/要点（可选）

Page N — 末页
├── 总结语（一句话）
├── CTA（收藏/关注引导）
└── 话题标签
```

### 封面布局规则
```
┌──────────────────────────────┐
│ 顶部标签：「教程」/「保姆级」    │
│                              │
│ 主标题：                      │
│   数字 + 动作 + 结果          │
│   例："3步搞定XX"             │
│   例："从0到1手把手教你XX"    │
│                              │
│ 副标题：                      │
│   限定范围/降低门槛           │
│   例："零基础也能学会"        │
│                              │
│ 步骤预告（可选）：             │
│   Step1 → Step2 → Step3      │
│   简化流程图                  │
│                              │
│ 底部：话题标签/品牌           │
└──────────────────────────────┘
```

### 步骤页布局规则
```
┌──────────────────────────────┐
│ 左上/顶部：步骤编号            │
│   超大数字（200-480px）        │
│   或：「Step N」标签           │
│                              │
│ 步骤标题：                    │
│   一个动作指令                │
│   48-72px, 粗体               │
│   例："打开终端"              │
│                              │
│ 步骤说明（≤2行）：            │
│   具体操作细节                │
│   28-40px                     │
│                              │
│ 代码块/截图/图标（可选）：     │
│   等宽字体 + 深色底           │
│   ≤3行代码                    │
│                              │
│ 底部：页码 N/Total            │
└──────────────────────────────┘
```

### 注意
- 每个步骤**只讲一件事**
- 步骤编号必须醒目（超大数字或带颜色标签）
- 如有代码，字号 ≥ 24px，等宽字体，深色底浅色字
- 步骤之间有明确的视觉分隔
- 进度感：通过页码或进度条暗示"还有多少步"

## 合集推荐型

### 适用场景
工具推荐、App推荐、书单、好物分享、旅行目的地

### 页面结构
```
Page 1 — 封面
├── 数字标题（"5个XX"）
├── 受众限定（"XX人必备"）
└── 亮点标签

Page 2~N-1 — 单品页（每页1-2个）
├── 编号
├── 名称（大字标题）
├── 一句话评价/亮点
├── 推荐理由（≤3条）
└── 标签/评分（可选）

Page N — 末页
├── 合集总结（一句话）
├── "还有哪些推荐？评论区见"
└── 话题标签
```

### 单品页布局规则
```
┌──────────────────────────────┐
│ 编号：大数字或圆形标签        │
│   01 / 02 / 03               │
│                              │
│ 名称/产品标题：               │
│   48-72px 粗体                │
│   例："Notion"               │
│                              │
│ 一句话亮点：                  │
│   20-28px 强调色              │
│   例："最强生产力工具"        │
│                              │
│ 推荐理由（要点列表）：         │
│   ✓ 理由1（≤15字）           │
│   ✓ 理由2（≤15字）           │
│   ✓ 理由3（≤15字）           │
│   28-36px，带图标前缀         │
│                              │
│ 底部：评分/标签/价格          │
└──────────────────────────────┘
```

### 注意
- 每页只放 1-2 个推荐项（不要堆砌）
- 编号要连续且醒目
- 推荐理由用短句+图标前缀（✓ 或 Lucide 图标）
- 如果是 App/工具，可放简化的 UI 示意或图标

## 对比型

### 适用场景
产品对比、使用前后对比、方案优劣分析、误区纠正

### 页面结构
```
Page 1 — 封面
├── 对比主题标题
├── "A vs B" 或 "错误 vs 正确"
└── 对比维度预告

Page 2~N-1 — 对比页
├── 对比维度标题
├── 左：A方案/旧做法/错误做法
├── 右：B方案/新做法/正确做法
└── 结论标签（推荐哪个）

Page N — 结论页
├── 总结推荐
├── 使用建议
└── CTA
```

### 对比页布局规则
```
┌──────────────────────────────┐
│ 维度标题（居中）：             │
│   "性能对比" / "价格对比"     │
│   48px 居中                   │
│                              │
│ ┌────────┐  ┌────────┐       │
│ │   ❌   │  │   ✅   │       │
│ │  旧/A  │  │  新/B  │       │
│ │        │  │        │       │
│ │ 描述   │  │ 描述   │       │
│ │ 28px   │  │ 28px   │       │
│ └────────┘  └────────┘       │
│                              │
│ 或上下排列：                  │
│ ━━━━ ❌ 错误做法 ━━━━         │
│   内容（淡红底/删除线）        │
│ ━━━━ ✅ 正确做法 ━━━━         │
│   内容（淡绿底/强调）          │
│                              │
│ 底部结论标签                  │
└──────────────────────────────┘
```

### 注意
- 对比必须有明确的视觉区分（颜色/位置/图标）
- ❌ 和 ✅ 使用 Lucide 图标 `x-circle` 和 `check-circle`（不是 emoji）
- 错误方案可用淡红底 + 删除线
- 正确方案用淡绿底 + 加粗
- 每页只对比 1-2 个维度

## 清单型

### 适用场景
注意事项、避坑指南、Do/Don't 列表、条件清单

### 页面结构
```
Page 1 — 封面
├── 清单标题（"N个XX注意事项"）
├── 重要性说明
└── 标签

Page 2~N-1 — 清单页（每页3-5条）
├── 条目编号
├── 条目内容
├── 图标前缀（✓/✗/⚠）
└── 补充说明（可选）

Page N — 末页
├── 总结
├── CTA
└── 标签
```

### 清单页布局规则
```
┌──────────────────────────────┐
│ 页面标题（可选）：             │
│   "第一部分：基础项"          │
│                              │
│ 清单条目（每条独立一行）：     │
│                              │
│  ✓  条目1文字（≤20字）       │
│     ──── 分隔线 ────          │
│  ✓  条目2文字（≤20字）       │
│     ──── 分隔线 ────          │
│  ✓  条目3文字（≤20字）       │
│     ──── 分隔线 ────          │
│  ✓  条目4文字（≤20字）       │
│                              │
│ 每条：                        │
│   图标(Lucide) + 文字         │
│   字号 28-36px                │
│   行间距 ≥ 60px（保证呼吸感）  │
│                              │
│ 底部：页码                    │
└──────────────────────────────┘
```

### 变体：Do/Don't 双列
```
┌──────────────────────────────┐
│        DO ✓         DON'T ✗   │
│  ┌──────────┐  ┌──────────┐  │
│  │ 做法1    │  │ 避免1    │  │
│  │ 做法2    │  │ 避免2    │  │
│  │ 做法3    │  │ 避免3    │  │
│  └──────────┘  └──────────┘  │
│   (绿色系底)    (红色系底)     │
└──────────────────────────────┘
```

### 注意
- 每条清单不超过 20 字
- 每页不超过 5 条（3-4 条最佳）
- 条目间用细线或间距分隔
- 图标前缀必须统一风格
- 重要条目可加底色高亮

## 金句/语录型

### 适用场景
人生感悟、名人名言、书摘金句、情感共鸣

### 页面结构
```
Page 1 — 封面
├── 主题标题或书名
├── 作者/来源
└── 标签

Page 2~N-1 — 金句页（每页1句）
├── 金句内容（大字居中）
├── 出处/作者
└── 装饰元素

Page N — 末页
├── 总结/推荐
├── CTA
└── 标签
```

### 金句页布局规则
```
┌──────────────────────────────┐
│                              │
│          大量留白             │
│                              │
│                              │
│    「                         │
│       金句内容               │
│       一行或两行             │
│       48-72px                │
│       居中对齐               │
│                」            │
│                              │
│     —— 作者名 ——             │
│        24px                  │
│                              │
│                              │
│          大量留白             │
│                              │
│ 底部：装饰元素/页码           │
└──────────────────────────────┘
```

### 注意
- 金句是唯一主角，**每页只放一句话**
- 字号大、居中、大量留白包围
- 可用引号装饰（「」或 " "）
- 风格推荐：guofeng（竖排）或 minimal（横排大字）
- 不加多余的装饰，让文字的力量说话

- 每页放 2-3 条，不要密集到看不清

## 数据报告型

### 适用场景
调查数据、行业报告、统计图表、百分比可视化

### 页面结构
```
Page 1 — 封面
├── 数据主题标题
├── 核心数字预告
└── 数据来源

Page 2~N-1 — 数据页
├── 数据点标题
├── 核心数字（超大装饰数字）
├── 数据解释（一句话）
└── 简化图表/进度条（可选）

Page N — 结论页
├── 数据洞察总结
├── 行动建议
└── CTA
```

### 数据页布局规则
```
┌──────────────────────────────┐
│ 数据标题：                    │
│   "关于XX的数据"             │
│   28-36px                    │
│                              │
│ 核心数字：                    │
│   超大装饰数字                │
│   200-480px                   │
│   强调色                      │
│   例："87%"                  │
│                              │
│ 数据解释：                    │
│   "XX的人选择了YY"           │
│   28-40px                    │
│                              │
│ 简化可视化（可选）：           │
│   进度条 / 圆环 / 对比柱      │
│   用 CSS 实现，不用 JS 图表   │
│                              │
│ 数据来源：                    │
│   20px 灰色文字               │
│   例："数据来源：XXX报告"    │
└──────────────────────────────┘
```

### 简化图表 CSS 实现方式
```html
<!-- 进度条 -->
<div style="height:12px;background:var(--border);border-radius:6px;overflow:hidden">
  <div style="width:87%;height:100%;background:var(--accent);border-radius:6px"></div>
</div>

<!-- 对比柱（水平） -->
<div style="display:flex;gap:8px;align-items:center">
  <div style="width:87%;height:32px;background:var(--accent);border-radius:8px"></div>
  <span style="font-size:28px;font-weight:700">87%</span>
</div>
```

### 注意
- 每页只讲 1 个数据点
- 数字是绝对主角（推荐 knowledge 风格的 big-num）
- 数据来源必须标注（增加可信度）
- 图表只用 CSS 实现（进度条/简单柱状图），不引入 Chart.js 等库
- 百分比 > 具体数字 > 倍数 > 文字描述（视觉冲击递减）

## 故事/经验型

### 适用场景
个人经历分享、踩坑总结、成长记录、转型经历

### 页面结构
```
Page 1 — 封面
├── 故事标题（制造好奇）
├── 时间跨度/结果预告
└── 标签

Page 2 — 背景/起因
├── 起点状态描述
├── 为什么开始
└── 核心矛盾/痛点

Page 3~N-2 — 经历/转折
├── 时间节点
├── 关键事件
├── 感悟/教训

Page N-1 — 结果/现状
├── 对比（before → after）
├── 核心收获

Page N — 末页
├── 一句话寄语
├── CTA
└── 标签
```

### 叙述页布局规则
```
┌──────────────────────────────┐
│ 时间节点标签：                │
│   "第3个月" / "2024年夏天"   │
│   20-24px 辅助色              │
│                              │
│ 事件标题：                    │
│   一句话概括当时经历          │
│   48-60px                    │
│                              │
│ 叙述内容（≤3行）：            │
│   具体细节 + 感受             │
│   28-32px                    │
│   不要写成长段落              │
│   每行一个信息点              │
│                              │
│ 金句/教训（可选）：            │
│   引用样式高亮                │
│   用边框/底色/引号区分正文     │
│                              │
│ 底部：页码/时间线指示          │
└──────────────────────────────┘
```

### 注意
- 不是写作文！每页只讲一个节点/事件
- 时间线感很重要（标注时间节点）
- 适合 minimal 或 guofeng 风格
- 可以用 before/after 对比增强说服力
- 结尾给出可操作的建议（不只是感慨）

## 测评/种草型

### 适用场景
产品真实使用体验、好物种草、对比测评

### 页面结构
```
Page 1 — 封面
├── 产品名 + 测评结论
├── 使用时长标注
└── 标签

Page 2 — 产品概览
├── 产品图/外观描述
├── 基本参数
└── 价格

Page 3~N-2 — 维度评分页
├── 评分维度名
├── 评分（星级/分数/进度条）
├── 该维度的体验描述
└── 优点/缺点标注

Page N-1 — 总结评分
├── 综合评分
├── 优缺点汇总
├── 适合/不适合谁

Page N — 末页
├── 购买建议
├── CTA
└── 标签
```

### 维度评分页布局规则
```
┌──────────────────────────────┐
│ 维度名：                      │
│   "拍照效果" / "续航表现"    │
│   48px 粗体                   │
│                              │
│ 评分可视化：                  │
│   ★★★★☆ (4/5)              │
│   或进度条 80%                │
│   或大数字 "8.5"             │
│                              │
│ 体验描述（≤3行）：            │
│   具体使用感受                │
│   28-32px                    │
│                              │
│ 优缺点标注：                  │
│   ✓ 优点（绿色/强调色）       │
│   ✗ 缺点（红色/灰色）         │
│                              │
│ 底部：页码                    │
└──────────────────────────────┘
```

### 注意
- 评分可视化必须有（星级/进度条/数字）
- 每个维度单独一页
- 优缺点用颜色+图标区分
- 最后的总结页可以放雷达图（CSS 实现）或汇总评分
- 推荐 food（实物产品）或 knowledge（数码产品）风格

## 使用方式

在阶段 2（知识文档 → 小红书内容）时：

1. **先判断内容类型**：根据素材内容确定属于上面哪种类型
2. **选择视觉风格**：参考推荐矩阵，或用户指定
3. **按内容类型结构编写页面脚本**：遵循对应类型的页面结构
4. **在阶段 3 生成 HTML 时**：内容结构（本文件）+ 视觉规范（style-templates.md）组合使用
